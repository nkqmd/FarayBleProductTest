# Android 产测 App UI 设计

## 1. 文档目标

本文用于补充 [架构设计.md](架构设计.md) 中的页面与交互设计章节，输出一份更适合直接落地开发的 UI 说明文档。目标是：

- 明确页面结构、核心组件和布局层级。
- 明确页面状态、按钮可用性和异常提示口径。
- 明确产测列表的展示规则、颜色规则和交互限制。
- 让后续 `Fragment + ViewModel + RecyclerView` 实现时有统一参照。

本文默认沿用当前项目技术形态：`Single-Activity + BottomNavigationView + Fragment + ViewModel + LiveData + Room`。

## 2. 设计原则

- 现场优先：界面优先展示“现在能不能测、测了多少、当前设备状态是什么”。
- 操作最短：每个页面只保留必要输入和必要按钮，减少误触。
- 状态清晰：重要状态通过“文案 + 颜色 + 禁用态”三种方式同时表达。
- 顺序稳定：产测列表只按首次出现顺序向下增长，不因状态变化重排。
- 安全优先：产测运行中锁定底部导航，必须先停止产测才能切页。
- 弱装饰风格：不追求复杂视觉效果，优先稳定、耐看、抗干扰。

## 3. 全局结构

## 3.1 页面结构

- 未登录：仅显示登录页。
- 已登录：进入业务容器页，底部导航包含 3 个 Tab：
  - 配置页
  - 产测页
  - 结果页

页面流转：

```text
登录页
  -> 配置页
     -> 产测页
     -> 结果页
```

约束：

- 登录页不显示底部导航栏。
- 底部导航栏只覆盖配置页、产测页、结果页。
- 产测运行中，底部导航栏显示但禁用点击切换。
- 产测停止后，恢复底部导航切换能力。

## 3.2 全局布局建议

业务页统一采用以下结构：

```text
+--------------------------------------------------+
| TopAppBar                                        |
| 页面标题 | 当前批次摘要/工位摘要                  |
+--------------------------------------------------+
| Fragment Content                                 |
| 页面核心内容                                     |
+--------------------------------------------------+
| BottomNavigationView                             |
| 配置 | 产测 | 结果                               |
+--------------------------------------------------+
```

TopAppBar 建议展示：

- 页面标题
- 当前批次 `batchId` 的简短显示
- 当前工位 `factoryId` 的简短显示

## 3.3 通用颜色规则

建议统一状态颜色，避免每页各自定义。

| 状态 | 颜色建议 | 用途 |
| --- | --- | --- |
| 默认/未开始 | `#ECEFF1` | 输入区、默认卡片背景 |
| 主操作 | `#1565C0` | 登录、确认、开始、上报按钮 |
| 进行中 | `#E3F2FD` + `#1565C0` | 加载中、测试中、排队中 |
| 成功 | `#E8F5E9` + `#2E7D32` | 通过、已完成成功 |
| 失败 | `#FFEBEE` + `#C62828` | 失败、错误 |
| 非法设备 | `#FFF3E0` + `#EF6C00` | 非法 MAC、非法广播 |
| 已产测 | `#F3E5F5` + `#6A1B9A` | 历史已产测设备 |
| 警告 | `#FFF8E1` + `#F9A825` | 重号、重复扫描提醒 |
| 禁用 | `#CFD8DC` + `#78909C` | 按钮不可点击 |

说明：

- 列表项背景色跟随设备状态变化。
- 颜色不能单独承担全部语义，必须同时显示状态文字。
- 若出现 `warning = DUPLICATE_MAC_IN_PROGRESS`，在原状态底色基础上加黄色描边或右上角警告标记。

## 3.4 通用文案规则

- 按钮文案尽量使用动词：`登录`、`确认并加载`、`开始产测`、`停止产测`、`结果上报`。
- 状态文案统一使用业务术语：
  - `待测`
  - `测试中`
  - `通过`
  - `失败`
  - `非法设备`
  - `已产测`
  - `重号警告`
- 成功率统一显示为百分比，例如 `92.31%`。
- 无数据时不显示空白区域，显示 `暂无数据`、`尚未开始产测`、`暂无上报记录` 等占位文案。

## 4. 登录页

## 4.1 页面目标

- 输入用户名和密码。
- 发起登录。
- 清晰反馈登录中、登录失败、登录过期等状态。

## 4.2 页面组成

- Logo 或页面标题
- 用户名输入框
- 密码输入框
- 登录按钮
- 登录中进度指示
- 错误提示区域

## 4.3 布局草图

```text
+--------------------------------------+
|           Android 产测 App           |
|            登录后进入业务页          |
|                                      |
| 用户名 [________________________]     |
| 密码   [________________________]     |
|                                      |
|          [   登录   ]                |
|                                      |
| 状态提示：账号或密码错误             |
+--------------------------------------+
```

## 4.4 交互规则

- 用户名、密码均非空时，登录按钮才可点击。
- 点击登录后：
  - 输入框禁用
  - 登录按钮进入 loading 态
  - 防止重复点击
- 登录成功后跳转到配置页。
- 登录失败后保留输入内容，展示错误文案。
- token 失效被动登出时，重新回到登录页，并显示一次性提示。

## 4.5 状态定义

| 状态 | 页面表现 |
| --- | --- |
| `Idle` | 输入框可编辑，按钮按正常态显示 |
| `Authenticating` | 输入框禁用，按钮显示 loading |
| `Authenticated` | 页面跳转到配置页 |
| `AuthExpired` | 显示“登录已失效，请重新登录” |
| `Error` | 显示具体错误文案 |

## 5. 导航页

这里的“导航页”不是独立业务页，而是登录成功后的业务容器结构。

## 5.1 导航结构

- 底部导航栏固定 3 个入口：
  - `配置`
  - `产测`
  - `结果`

建议图标：

- 配置：设置或下载图标
- 产测：蓝牙或播放图标
- 结果：列表或统计图标

## 5.2 导航规则

- 默认登录成功后进入配置页。
- 配置完成前，允许切到产测页，但应显示“请先完成批次配置”的空状态提示，并禁用开始产测按钮。
- 产测运行中：
  - 底部导航栏整体置灰
  - 当前页停留在产测页
  - 用户只能点击“停止产测”
- 产测停止后：
  - 可切换到结果页查看本次会话统计
- 登出后：
  - 销毁业务容器状态
  - 返回登录页

## 6. 配置页

## 6.1 页面目标

- 输入批次号。
- 点击确认后顺序执行：
  - 摘要获取
  - MAC 表下载
- 展示配置结果和当前批次摘要。
- 为产测页提供可执行的批次上下文。

## 6.2 页面组成

- `factoryId` 输入框
- `batchId` 输入框
- `确认并加载` 按钮
- 结果显示区域
- 摘要卡片
- MAC 导入结果卡片

## 6.3 布局草图

```text
+--------------------------------------------------+
| 配置                                             |
| 工位号   [____________________]                  |
| 批次号   [____________________]                  |
|                                                  |
|            [ 确认并加载 ]                        |
|                                                  |
| 执行结果                                         |
| - 摘要获取：成功 / 失败                          |
| - MAC表下载：成功 / 失败                         |
| - 导入数量：100000                               |
| - 最近更新时间：2026-04-09 10:30                 |
|                                                  |
| 当前批次摘要                                     |
| - 批次号：BATCH20260409001                       |
| - 设备型号：xxx                                  |
| - 目标数量：1000                                 |
| - 固件版本：1.0.3                                |
| - 广播前缀：CL-xxf_                              |
+--------------------------------------------------+
```

## 6.4 交互规则

- `factoryId` 和 `batchId` 非空时，`确认并加载` 按钮才可点击。
- 点击 `确认并加载` 后，按固定顺序执行：
  1. 获取批次摘要
  2. 下载 MAC 表
  3. 导入数据库
  4. 更新页面摘要和结果显示
- 任一步失败，都要停止后续步骤并显示失败原因。
- 配置成功后：
  - 页面显示“批次已就绪”
  - 产测页允许点击“开始产测”
- 如果当前存在运行中的产测会话，不允许重新加载新批次。

## 6.5 结果显示规则

结果显示区建议分为两块：

- 执行状态
  - `摘要获取成功`
  - `MAC表下载成功`
  - `导入完成，共 N 条`
- 错误信息
  - 网络失败
  - 批次不存在
  - MAC 导入中断
  - 配置字段非法

## 6.6 页面状态

| 状态 | 页面表现 |
| --- | --- |
| `Idle` | 允许输入，结果区为空 |
| `LoadingBatch` | 按钮 loading，输入禁用 |
| `ImportingMacList(progress)` | 显示进度条和导入数量 |
| `Ready` | 显示摘要卡片，允许开始产测 |
| `Error` | 显示失败文案，保留输入值 |

## 7. 产测页

## 7.1 页面目标

- 启动/停止产测。
- 实时展示统计信息。
- 按首次出现顺序展示 DUT 列表。
- 通过背景色和状态文案直观表达测试结果。

## 7.2 页面组成

- 顶部运行控制区
  - `开始产测` 按钮
  - `停止产测` 按钮
  - 当前批次简要信息
- 统计信息卡片
  - 测试总数
  - 实测数
  - 成功数
  - 失败数
  - 非法数
  - 成功率
- DUT 列表
  - `mac`
  - `rssi`
  - `状态`

## 7.3 布局草图

```text
+--------------------------------------------------+
| 产测                                             |
| 批次：BATCH20260409001   工位：F01               |
|                                                  |
| [ 开始产测 ]   [ 停止产测 ]                      |
|                                                  |
| 统计信息                                         |
| 总数  120 | 实测数 100 | 成功数 95               |
| 失败数 3  | 非法数 2   | 成功率 95.00%           |
|                                                  |
| DUT 列表                                         |
| ------------------------------------------------ |
| MAC              RSSI      状态                  |
| 010203040506     -48       通过                  |
| 010203040507     -61       失败                  |
| 010203040508     -58       非法设备              |
| 010203040509     -52       测试中                |
| ...                                              |
+--------------------------------------------------+
```

## 7.4 按钮规则

| 场景 | 开始产测 | 停止产测 |
| --- | --- | --- |
| 未完成配置 | 禁用 | 禁用 |
| 已完成配置，未运行 | 可点击 | 禁用 |
| 产测运行中 | 禁用 | 可点击 |
| 停止中 | 禁用 | loading / 禁用 |
| 已停止 | 可重新开始新会话 | 禁用 |

## 7.5 统计口径

建议口径如下：

- 测试总数：当前会话内进入结果视图的设备总数，建议为 `成功数 + 失败数 + 非法数 + 仍在测试中的合法设备数`
- 实测数：已真正进入 BLE 测试流程的数量，通常为 `成功数 + 失败数 + 测试中的数量`
- 成功数：最终 `PASS`
- 失败数：最终 `FAIL`
- 非法数：`INVALID_DEVICE`
- 成功率：`成功数 / max(实测数, 1)`，无实测时显示 `0.00%`

说明：

- 如果产品侧后续要求“测试总数”改成“成功 + 失败 + 非法”的最终值，可以在 ViewModel 中单独调整，但 UI 字段名不变。

## 7.6 DUT 列表规则

列表字段固定为：

- `mac`
- `rssi`
- `状态`

可选扩展字段：

- 最近出现时间
- 广播名
- 警告标记

展示规则：

- 首次扫描到时分配 `sequenceNo`。
- 新设备追加到列表底部。
- 设备状态变化时只刷新当前项，不重排。
- 列表按 `sequenceNo ASC` 展示。
- 页面滚动位置不因局部刷新跳动。

## 7.7 列表项颜色规则

| 状态 | 背景色 | 前景色 | 文案 |
| --- | --- | --- | --- |
| `DISCOVERED` | `#ECEFF1` | `#37474F` | 已发现 |
| `QUEUED` | `#E3F2FD` | `#1565C0` | 待测 |
| `CONNECTING` / `SUBSCRIBING` / `SENDING` / `WAITING_NOTIFY` / `DISCONNECTING` | `#E1F5FE` | `#0277BD` | 测试中 |
| `PASS` | `#E8F5E9` | `#2E7D32` | 通过 |
| `FAIL` | `#FFEBEE` | `#C62828` | 失败 |
| `INVALID_DEVICE` | `#FFF3E0` | `#EF6C00` | 非法设备 |
| `ALREADY_TESTED` | `#F3E5F5` | `#6A1B9A` | 已产测 |

附加警告：

- `DUPLICATE_MAC_IN_PROGRESS`：在当前状态基础上增加黄色描边、警告角标或 `重号` 标签。

## 7.8 页面状态与空态

| 状态 | 页面表现 |
| --- | --- |
| 未配置 | 显示“请先在配置页完成批次加载” |
| 已配置未开始 | 开始按钮可用，列表为空 |
| 运行中 | 更新统计卡片和设备列表，禁用导航切换 |
| 停止中 | 停止按钮 loading，保留当前列表 |
| 已停止 | 列表冻结，允许切换到结果页 |
| 异常 | 顶部显示错误横幅，但不丢失已展示的历史项 |

## 7.9 关键交互限制

- 运行中不能切换到底部导航其他页面。
- 运行中不能重新配置批次。
- 停止产测后，等待在途任务结束或超时，再把会话标记为 `STOPPED`。
- 只有 `STOPPED` 后，结果页的“结果上报”按钮才允许点击。

## 8. 结果页

## 8.1 页面目标

- 展示当前最近一个已停止会话的统计结果。
- 允许手动触发本次会话结果上报。
- 展示上报结果和服务端回显。

## 8.2 页面组成

- 会话摘要卡片
- 统计信息卡片
- `结果上报` 按钮
- 上报结果显示区

## 8.3 布局草图

```text
+--------------------------------------------------+
| 结果                                             |
| 当前会话：SESSION_20260409_001                   |
| 批次：BATCH20260409001                           |
|                                                  |
| 统计信息                                         |
| - 测试总数：120                                  |
| - 实测数：100                                    |
| - 成功数：95                                     |
| - 失败数：3                                      |
| - 非法数：2                                      |
| - 成功率：95.00%                                 |
|                                                  |
|             [ 结果上报 ]                         |
|                                                  |
| 上报结果                                         |
| - 状态：上报成功                                 |
| - upload_id：UPL_20260409_0001                   |
| - duplicate：false                               |
| - message：服务端已接收                          |
+--------------------------------------------------+
```

## 8.4 按钮规则

- 以下情况禁用 `结果上报`：
  - 当前没有已停止会话
  - 当前会话仍在运行
  - 正在上报中
- 点击后进入 loading 态，防止重复点击。
- 上报完成后保留最近一次结果显示。

## 8.5 结果显示规则

结果显示区建议至少展示：

- 上报状态：成功 / 失败 / 重复上报
- 服务端返回消息
- `upload_id`
- 上报时间

若服务端返回 `duplicate = true`：

- 仍视为业务成功
- 文案显示为 `重复上报，服务端已存在该结果`

## 8.6 页面状态

| 状态 | 页面表现 |
| --- | --- |
| `Empty` | 显示“暂无可上报结果” |
| `Ready` | 显示统计信息，允许点击上报 |
| `Uploading` | 按钮 loading，页面其余区域只读 |
| `UploadSuccess` | 显示成功回显 |
| `UploadDuplicate` | 显示重复上报提示 |
| `UploadError` | 显示错误文案，允许重试 |

## 9. 页面实现映射建议

为方便直接落地到 Android 页面层，建议采用以下对应关系：

| 页面 | Fragment | ViewModel | 主要列表/组件 |
| --- | --- | --- | --- |
| 登录页 | `LoginFragment` | `AuthViewModel` | 输入框、按钮、状态提示 |
| 配置页 | `ConfigFragment` | `ConfigViewModel` | 摘要卡片、导入结果区 |
| 产测页 | `ProductionFragment` | `ProductionViewModel` | 统计卡片、`RecyclerView` |
| 结果页 | `ResultFragment` | `ResultViewModel` | 统计卡片、上报结果区 |

## 10. UI 验收口径

- 登录页必须支持用户名、密码输入和登录反馈。
- 登录成功后必须进入带底部导航的业务容器。
- 配置页必须支持批次输入、确认加载、摘要获取、MAC 表下载结果显示。
- 产测页必须支持开始/停止按钮、6 项统计信息和 DUT 列表展示。
- DUT 列表必须按首次出现顺序向下增长，状态更新不重排。
- DUT 列表背景色必须跟随状态变化，至少区分通过、失败、非法、已产测、测试中。
- 产测运行中必须锁定底部导航切换。
- 结果页必须支持统计展示、结果上报和上报结果回显。

## 11. 后续实现建议

- 优先实现低保真 XML 布局，不必一开始就做复杂视觉样式。
- 先把状态颜色、禁用逻辑、列表刷新规则做正确，再细化间距和字体。
- `ProductionFragment` 的 `RecyclerView.Adapter` 必须支持按 `mac` 或 `sequenceNo` 做局部更新，避免整表闪烁。
- 统计卡片建议做成独立可复用组件，保证配置页、产测页、结果页风格统一。
